Introduction & Setup

Seeking Localized Intelligence

Khoji is a local-first, offline AI knowledge operating system that transforms multi-format documents (PDFs, images, and epubs) into a permanent, interconnected knowledge graph. Built with PySide6 for high-performance desktop execution, Khoji guarantees that all document reading, layout parsing, concept extraction, vector embedding, and LLM reasoning remain completely on your local device.

Quick Start Setup

Follow these CLI commands to initialize a virtual environment, install local dependencies, and spin up the Khoji Desktop application.

            

              

                
                
                
              

              Terminal - Setup
              
            

            # Navigate to workspace
cd khoji_desktop

# Create virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies (with OCR fallbacks and dev tools)
pip install -e ".[ocr,dev]"

# Run Khoji Desktop
python -m khoji.main

Product Requirements Specifications (PRD)

Khoji is built to solve fragmented information mastery. By running completely on-device, it provides researchers, engineers, and students a private second brain that reads materials, extracts concepts, schedules active recall drills, and responds contextually.

Feature Priorities

P0 — Core Essentials (MVP Release)

Multi-Format Ingestion

Drag-and-drop parsing of PDF files up to 200MB. PyMuPDF parsing and RapidOCR scanning fallback.

Multi-Agent pipeline

Orchestrates isolated models to run OCR, layout structure analysis, concept indexing, summaries, and quizzes.

Spaced Repetition

Automatic cloze and Q&A flashcard generation reviewed through an FSRS or SM-2 algorithm schedule.

Semantic Search

Global vector search powered by local sentence-transformers (MiniLM) and FAISS indexes.

P1 — Secondary Expansion

Collections

Folder systems and smart query rules grouping files automatically based on tags (e.g. #quantum).

Mind Maps & Timelines

Automatically maps chronological milestones and structural links into zoomable Mermaid.js canvases.

AI Memory Customization

Saves preferences (explanations detail, analogies choice) and corrections to shape future LLM reasoning.

P2 — Future Roadmap

Plugin architecture supporting custom parsers and export adapters; peer-to-peer LAN knowledge synchronization; local Git integrations tracking changes to markdown notes; adaptive tutoring schedules.

Multi-Agent AI Pipeline

Rather than overloading a single model with raw PDF content, Khoji orchestrates a network of 12 specialized agents. Each agent processes, translates, indexes, or formats document sections contextually.

Model Allocation & Hardware Scaling Strategy

To support machines ranging from 4GB RAM to multi-GPU workstations, the pipeline orchestrator automatically detects hardware presets and maps tasks to smaller, optimized local models.

Pipeline Task Low RAM Setup (4GB RAM) Medium Setup (8GB RAM) High RAM Setup (16GB+ RAM)
OCR Tesseract v5 (Native C++) EasyOCR (PyTorch CPU) Surya OCR (GPU accelerated)
Layout Segmentation Rule-based coordinates geometry LayoutLMv3 (Lightweight model) LayoutLMv3 (Full segmentation)
Vision Descriptions Skipped (Degrades gracefully) Florence-2 Base (ONNX) Florence-2 Large (FP16 GPU)
Embeddings Generation all-MiniLM-L6-v2 (384-dim) all-MiniLM-L6-v2 (384-dim) BGE-Large (1024-dim)
Reasoning & Generation Llama-3.2-1B (Q4_K_M GGUF) Phi-3-Mini (3.8B Q4 GGUF) Mistral-7B-Instruct (Q8 GGUF)
Quiz & Card Prompts Llama-3.2-1B (Strict Template) Phi-3-Mini (Structured JSON) Mistral-7B (Autonomous Reasoning)

Database Schema & Layout Design

Khoji manages structured relational data in a local SQLite database (`~/.khoji/db/khoji.db`) running in Write-Ahead Logging (WAL) mode. High-dimensional vector search mappings are resolved using the sqlite-vec SQLite extension compiled locally.

    Accessibility & Performance Specifications

    Khoji Desktop targets WCAG 2.1 AA accessibility guidelines and sets strict local performance bounds to run comfortably on low-end hardware.

    Accessibility Checklist (WCAG 2.1 AA)

    Contrast Targets

    Maintains WCAG AAA contrast ratios (minimum 4.5:1, targets 7+:1) for all typography elements across light and dark theme configurations.

    Keyboard Traps

    Ensures tab cycles are closed within overlays; allows skip-to-content links and exposes 2px visible primary focus rings around active inputs.

    Screen Readers

    All PySide6 UI widgets assign standard ARIA label mappings and leverage QAccessible interfaces for screen reader compatibility.

    Motion Reduction

    Supports prefers-reduced-motion parameters, disabling page flips or loading animations upon system request.
